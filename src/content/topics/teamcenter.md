# Siemens Teamcenter

## 概要

Siemens Teamcenterは、Siemens Digital Industries Softwareが提供する製品ライフサイクル管理（PLM: Product Lifecycle Management）ソフトウェアです。製品の企画・設計・製造・保守に至るまでのすべてのデータを一元管理します。

## 基本概念

### PLM（製品ライフサイクル管理）とは

製品のライフサイクル全体を通じて、製品に関連するすべての情報を管理するプロセスおよびシステム。

**管理対象データの例：**
- CADデータ（設計図面、3Dモデル）
- BOM（部品表: Bill of Materials）
- 技術仕様書
- 変更履歴
- 承認ワークフロー
- サプライヤー情報

### Teamcenterの主要機能

| 機能 | 説明 |
|------|------|
| **データ管理** | CADファイル、ドキュメント、BOMの一元管理 |
| **ワークフロー管理** | 承認プロセス、変更管理の自動化 |
| **コラボレーション** | 部門間・拠点間でのリアルタイム協業 |
| **バージョン管理** | 設計データの履歴管理、リビジョン追跡 |
| **アクセス制御** | ユーザー権限に基づくデータアクセス管理 |

│                  クライアント層                    │

└─────────────────────────────────────────────────┘
                        │
                        ▼
┌─────────────────────────────────────────────────┐
│                 アプリケーション層                 │
│          (Teamcenter Server / AWC Server)       │
└─────────────────────────────────────────────────┘
                        │
                        ▼
┌─────────────────────────────────────────────────┐
│                   データ層                       │
│        (Database / File Management System)      │
└─────────────────────────────────────────────────┘

### サーバー構成要素

- **Teamcenter Server**: コアビジネスロジックを処理
- **File Management System (FMS)**: 大容量ファイル（CADデータ等）の保存・転送
- **Database Server**: メタデータ、構造情報の保存（Oracle、SQL Server等）

## Multi-Site Collaboration（マルチサイトコラボレーション）

### マルチサイト構成とは

地理的に分散した複数の拠点（サイト）間でPLMデータを共有・同期するためのアーキテクチャ。グローバル企業での製品開発に不可欠な機能です。

### コーポレートサーバー（Corporate Server）

**コーポレートサーバー**は、マルチサイト構成における中核的なサーバーで、以下の役割を持ちます：

| 役割 | 説明 |
|------|------|
| **中央集権的なデータ管理** | 企業全体のPLMデータのマスターコピーを保持 |
| **データオーナーシップ** | 特定のデータオブジェクトに対する「所有権」を管理 |
| **複数サイト間の同期** | サイトサーバーへのデータレプリケーション・配布 |
| **グローバルコラボレーション** | 世界中の拠点間での安全なデータ共有 |

### マルチサイト構成の例

```
┌─────────────────────────────────────┐
│     コーポレートサーバー（本社）       │
│     （マスターデータの所有者）         │
└──────────────┬──────────────────────┘
               │
    ┌──────────┼──────────┐
    │          │          │
    ▼          ▼          ▼
┌───────┐  ┌───────┐  ┌───────┐
│サイトA │  │サイトB │  │サイトC │
│(欧州)  │  │(北米)  │  │(アジア)│
└───────┘  └───────┘  └───────┘
  レプリカ   レプリカ   レプリカ
```

### データオーナーシップ

マルチサイト環境では、各データオブジェクトには「所有サイト」が設定されます：

- **所有サイト**: データの変更権限を持つ唯一のサイト
- **レプリカサイト**: データの読み取り専用コピーを保持
- **オーナーシップ転送**: 必要に応じて所有権を別サイトに移転可能

```
例：製品Aの設計データ
├── 所有サイト: 本社（日本）→ 編集可能
├── レプリカ: 北米工場 → 参照のみ
└── レプリカ: 欧州工場 → 参照のみ
```

### レプリケーションの種類

| 種類 | 説明 | 用途 |
|------|------|------|
| **プッシュ型** | 所有サイトから他サイトへ自動配布 | 設計完了後の展開 |
| **プル型** | 必要なサイトがデータを要求 | オンデマンドでのデータ取得 |
| **同期型** | リアルタイムでの双方向同期 | 緊密なコラボレーション |

### マルチサイトのメリット

1. **ネットワーク遅延の軽減**: 各地域のユーザーはローカルサーバーにアクセス
2. **高可用性**: 一部サイトが停止しても他サイトで業務継続可能
3. **帯域幅の最適化**: 差分同期によりネットワーク負荷を軽減
4. **セキュリティ**: 地域ごとのアクセス制御とデータ分離

## クライアントの種類

### Rich Client

- Windowsデスクトップアプリケーション
- 高機能で複雑な操作に対応
- CAD統合機能が充実

### Active Workspace Client (AWC)

- Webブラウザベースのモダンなクライアント
- レスポンシブデザインでモバイル対応
- 直感的なUI/UX

### Visualization

- 3Dビューア機能
- CADデータの閲覧・マークアップ
- 設計レビューに使用

## 関連用語

### Item / Item Revision

- **Item**: 管理対象物（部品、製品、ドキュメント等）
- **Item Revision**: Itemの特定バージョン（リビジョン）

### Dataset

CADファイルやドキュメントなど、実際のファイルを格納するオブジェクト。

### BOM View Revision (BVR)

製品構造（部品表）のリビジョン管理されたビュー。

### Workflow

承認プロセスや変更管理を自動化するためのワークフロー定義。

## トラブルシューティング

### よくある問題

| 問題 | 原因 | 対処法 |
|------|------|--------|
| ログインできない | ライセンス不足、サーバー停止 | ライセンス状況・サーバー状態を確認 |
| ファイルが開けない | FMSサーバー障害 | FMSサーバーの状態を確認 |
| 同期が遅い | ネットワーク帯域不足 | 同期スケジュールの調整 |
