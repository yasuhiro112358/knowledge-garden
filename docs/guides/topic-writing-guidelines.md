# トピック作成ガイドライン

このドキュメントは、Knowledge Gardenのトピック（`src/content/topics/`）を作成する際の方針とルールを定めています。

## 基本方針

### 概念的な説明にとどめる

トピックでは、技術やソフトウェアの**概念的な説明**を記載します。具体的な技術仕様や設定方法は記載しません。

**良い例：**
- 「〇〇とは何か」という概念の説明
- 用語の一般的な意味
- 技術の位置づけや役割

**避けるべき例：**
- 具体的なアーキテクチャ図や構成要素の詳細
- 設定ファイルの記述方法
- バージョン固有の仕様
- トラブルシューティングの詳細手順

### 担保できない情報は記載しない

情報の正確性を確認できない場合は、記載しません。

- 公式ドキュメントで確認できない情報
- NDAや契約上の制約がある情報
- バージョンによって異なる可能性がある詳細仕様

### 情報源について明記する

トピックの末尾に「注意事項」セクションを設け、以下を明記します：

- このページの内容は概念的な説明にとどめていること
- 詳細は公式ドキュメントを参照するよう案内すること

## トピックの構成

### 推奨構成

```markdown
# トピック名

## 概要

このトピックが何であるかを1〜2文で説明。

## 〇〇とは

概念的な説明を記載。

## 注意事項

このページの内容は概念的な説明にとどめています。
具体的な仕様については、公式ドキュメントを参照してください。
```

### 記載量の目安

- 20〜50行程度
- 長くなりすぎる場合は、情報を削減することを検討

## 対象となるトピック

このガイドラインは、特に以下のようなトピックに適用されます：

- 商用ソフトウェア（Siemens Teamcenterなど）
- エンタープライズ製品
- 公開情報が限られている技術
- NDAや契約上の制約がある製品

## 例

### Siemens Teamcenter の例

```markdown
# Siemens Teamcenter

## 概要

Siemens Teamcenterは、Siemens Digital Industries Software（DISW）が
提供する製品ライフサイクル管理（PLM）ソフトウェアです。

## PLM（製品ライフサイクル管理）とは

PLM（Product Lifecycle Management）は、製品の企画から設計、製造、
保守、廃棄に至るまでのライフサイクル全体を通じて、製品に関連する
データやプロセスを管理するための考え方・システムです。

## コーポレートサーバーとは

Teamcenterには「マルチサイト」と呼ばれる構成があり、地理的に分散した
複数の拠点間でデータを共有・同期することができます。

**コーポレートサーバー（Corporate Server）**は、このマルチサイト構成に
おいて中核となるサーバーを指す用語です。

## 注意事項

このページの内容は概念的な説明にとどめています。
Teamcenterの具体的な仕様については、Siemensの公式ドキュメントを
参照してください。
```
