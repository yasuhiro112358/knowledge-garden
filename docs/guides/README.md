## ドキュメントの分類

### 手順書（Guides）
- **目的**: 実際の作業手順を説明
- **対象**: 開発者・運用者
- **内容**: ステップバイステップの手順、コマンド例、トラブルシューティング

### 仕様書・設計書（Specifications）
- **目的**: システムの仕様や設計を説明
- **対象**: 開発者・設計者
- **内容**: 要件定義、機能仕様、技術選定、アーキテクチャ決定

### 参考資料（References）
- **目的**: 技術的な参考情報を提供
- **対象**: 開発者
- **内容**: 技術比較、提案書、プロジェクト履歴

## 手順書の更新方針

- 手順書は実際の作業手順に基づいて記述する
- 手順が変更された場合は、すぐに更新する
- 古くなった手順書は削除またはアーカイブする
